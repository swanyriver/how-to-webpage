Success And Failure

Before we continue automating git repository creation we are going to want to be able to tell if our curl calls were successful. That way we will be able to branch on this and display an error message if it was unsuccessful or perform more operations like setting up our local git to track changes from the remote if it was successfully created.

The curl call will return an exit status but what we are more interested in is the HTTP code returned from the github server. To store this code in a BASH variable we will use command substitution, allowing the normally printed output of our command to be stored in a variable. and that takes the form $(OUR CURL CALL). We will set the output to be only the http response code by setting the --write-out flag to http_code, but we will also need to redirect the returned JSON data using the --output flag. This can be stored in file for later retrieval but for now we will send it to /dev/null

defining a few variables ahead of time and using command line arguments to fill in the name and description of our new repository our new curl call:

auth=$(USERNAME:PASSWORD)
json="{\"name\":\"$1\", \"description\":\"$2\"}"
url="https://api.github.com/user/repos"
htresp=$(curl --write-out %{http_code} --output /dev/null -u "$auth" $url -d "$json" )
